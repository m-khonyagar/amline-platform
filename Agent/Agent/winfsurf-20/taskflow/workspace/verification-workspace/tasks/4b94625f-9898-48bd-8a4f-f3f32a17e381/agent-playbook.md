# Agent Improvement Playbook

Continuously improve by reviewing recent runs, extracting reusable lessons, and feeding a compact playbook into the next agent workflow.

## Active Guidance
- Prefer short, reversible execution bursts when a request needs live operator actions.
- Create a visible artifact early and keep the task summary concise so users can verify outcomes quickly.
- Prefer the most relevant operational tools first: telegram, bale, workflow_task, gmail.

## Preferred Tools
- telegram, bale, workflow_task, gmail, file_share
