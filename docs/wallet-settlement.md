# کیف پول و تسویه (طرح)

## اهداف

- **Ledger** دوطرفه (بدهکار/بستانکار) با `idempotency_key` برای هر تراکنش درگاه.
- گزارش **تسویه دوره‌ای** (روزانه/هفتگی) قابل audit.

## وضعیت

منطق فعلی در مونولیت؛ settlement خودکار در این مخزن هنوز پیاده‌سازی کامل نشده.

## گام بعد

1. جدول `ledger_entries` + invariant «جمع صفر در هر تراکنش منطقی».
2. webhook درگاه → worker تأیید و ثبت idempotent.
3. API ادمین: export CSV و وضعیت تسویه.
