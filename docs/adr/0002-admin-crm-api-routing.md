# ADR 0002 — مسیریابی API مدیریت CRM

## وضعیت

پذیرفته‌شده — 2026-04

## زمینه

دو مسیر مرتبط با «CRM» در اپ وجود داشت: یکی زیر پیشوند `/admin/crm` برای پنل مدیریت و یکی زیر `/api/v1` با پیاده‌سازی‌های متفاوت. بدون ترتیب صحیح mount در `app.main`، درخواست‌های `/admin/crm/*` ممکن بود به router سبک‌تر یا in-memory بروند و تست‌ها یا UI ادمین 404 می‌گرفتند.

## تصمیم

1. **`admin_panel.router`** با **`prefix="/admin"`** روی **`app`** include شود و **قبل از** mount کردن `platform_router` (یا هر mount ریشه‌ای که مسیرهای CRM جایگزین را سرو می‌کند) قرار گیرد تا **`/admin/crm/*`** همیشه به **هندلرهای DB-backed** در `app/api/routes/admin_panel.py` برسد.

2. مسیرهای legacy یا نمونهٔ in-memory تحت **`/api/v1/...`** (در صورت باقی ماندن) صرفاً برای سازگاری عقب‌تر یا توسعه جدا نگه داشته می‌شوند؛ **منبع حقیقت دادهٔ CRM برای ادمین** همان مدل‌های P1 در `app/models/crm.py` و جداول مربوط است.

3. تست‌هایی که به **Redis Streams** (`XADD`، `XREAD`، `XGROUP` و غیره) وابسته‌اند با **`@pytest.mark.redis`** علامت می‌خورند؛ اجرای محلی بدون Redis سازگار: **`pytest --skip-redis`**.

## پیامدها

- ترتیب include در `main.py` بخشی از قرارداد است؛ تغییر ترتیب می‌تواند دوباره مسیر اشتباه را فعال کند.
- CI یا محیط توسعه باید یا Redis 5+ با Streams داشته باشد یا در pipeline از `--skip-redis` استفاده شود و تست‌های redis را در job جدا اجرا کند.
