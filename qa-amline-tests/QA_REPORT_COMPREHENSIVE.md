## Executive Summary

این گزارش نتیجه تست جامع اپلیکیشن تنظیم قرارداد رهن و اجاره املاین است. تست‌ها با شبیه‌سازی رفتار کاربران واقعی در سناریوهای مختلف انجام شده و شامل موارد زیر است:

- تست‌های پایه (Happy Path) برای نقش‌های مالک، مستاجر و تنظیم‌کننده
- تست‌های منفی برای تمام فیلدها
- اعتبارسنجی منطقی بین فیلدها
- سناریوهای مرزی (Edge Cases)
- ارزیابی تجربه کاربری (UX)

## ۱. دسته‌بندی مشکلات

### ۱.۱ ورود و احراز هویت (OTP)

- اعتبارسنجی شماره موبایل (کمتر/بیشتر از ۱۱ رقم، خالی، فرمت اشتباه)
- اعتبارسنجی کد OTP
- پیام‌های خطا و وضوح آن‌ها
- رفتار در صورت ارسال مجدد کد

### ۱.۲ اعتبارسنجی فیلدها

- کد ملی (۱۰ رقم، فرمت، حروف)
- شماره تماس
- آدرس و کد پستی
- مبالغ (اجاره، ودیعه)
- تاریخ‌ها
- متراژ ملک
- شماره شبا

### ۱.۳ منطق قرارداد

- رابطه تاریخ شروع و پایان
- تاریخ تحویل ملک
- مبالغ منطقی (مثبت، غیرصفر)
- سازگاری اطلاعات طرفین (کد ملی و موبایل متعلق به یک نفر)

### ۱.۴ تجربه کاربری (UX)

- وضوح مسیر کاربر
- پیام‌های خطا قابل فهم
- امکان بازگشت به مراحل قبل
- پشتیبانی RTL
- فیلدهای readonly (تاریخ تولد) نیاز به کلیک برای باز شدن دارند

## ۴. پیشنهادات بهبود

### ۴.۱ اعتبارسنجی

1. **اعتبارسنجی یکپارچه:** استفاده از Zod یا مشابه در frontend و Pydantic در backend با قوانین یکسان.
2. **شماره موبایل:** اعتبارسنجی سخت‌گیرانه قبل از ارسال: `^09[0-9]{9}$`، `maxLength=11`.
3. **کد ملی:** ۱۰ رقم عددی، الگوریتم کنترل اعتبار.
4. **مبالغ:** `min > 0` برای اجاره و ودیعه.
5. **تاریخ‌ها:** پایان ≥ شروع، تحویل ≥ شروع.

### ۴.۲ تجربه کاربری

1. **پیام‌های خطا:** متن‌های مشخص و راهنما برای هر فیلد.
2. **فیلدهای تاریخ:** امکان تایپ مستقیم یا راهنمای واضح برای date picker.
3. **ذخیره پیش‌نویس:** امکان ذخیره موقت فرم برای جلوگیری از از دست رفتن داده در رفرش.
4. **جلوگیری از کلیک چندباره:** غیرفعال کردن دکمه در حین ارسال درخواست.

### ۴.۳ تست خودکار

1. اجرای منظم Playwright در CI/CD.
2. تست‌های منفی برای تمام فیلدها.
3. تست منطق بین فیلدها (تاریخ، مبالغ، سازگاری).

## ۵. نحوه استفاده از این گزارش

1. **برای Google Docs:** محتوای این فایل را کپی و در Google Docs قرار دهید.
2. **تست‌های Playwright:** `cd qa-amline-tests && npm test`
3. **تست OTP دستی:** `npm run test:otp` و در لحظه توقف، کد را وارد کنید.
4. **مرورگر زنده:** `cd browser-live-control && npm run start-chrome` سپس `node run-command.js <command>`
5. برای هر باگ جدید، یک ردیف در بخش «لیست کامل باگ‌ها» اضافه کنید.
