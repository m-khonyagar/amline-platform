## باگ کشف‌شده

### باگ: تاریخ تولد معتبر شناخته نمی‌شود

| فیلد | مقدار |
|------|-------|
| **عنوان** | فیلدهای تاریخ تولد (روز/ماه/سال) مقدار را به API ارسال نمی‌کنند |
| **پیام خطا** | `Input should :('natural_person_detail', 'birth_date be a valid date` |
| **علت احتمالی** | فیلدها `readonly` هستند و با تایپ مستقیم، state فرم به‌روز نمی‌شود |
| **شدت** | زیاد |
| **پیشنهاد** | استفاده از date picker واقعی یا اطمینان از به‌روزرسانی state با تغییر value |

---

## موانع و نکات

1. **Toast خطا:** نوتیفیکیشن‌های خطا (Sonner) روی دکمه «مرحله بعد» قرار می‌گیرند و کلیک را مسدود می‌کنند. با مخفی کردن موقت toast، کلیک انجام شد.

2. **فیلدهای readonly:** فیلدهای تاریخ تولد readonly هستند و نیاز به باز کردن date picker دارند.

3. **اعتبارسنجی سمت سرور:** کد ملی و موبایل باید متعلق به یک نفر باشند؛ داده‌های تست ممکن است در محیط واقعی رد شوند.
